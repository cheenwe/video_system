# V视频站 — 系统使用说明手册

本手册面向**首次使用本系统**的观众、UP 主与管理员。按「系统是什么 → 各页面做什么 → 不同角色怎么用」编写；会使用浏览器即可，不要求编程基础。

---

## 一、系统概述

### 1.1 系统定位

本系统是一套可自部署的 **视频点播与管理平台**，在浏览器中运行，用于：

- **前台**：浏览公开/隐私视频、播放、点赞收藏、评论、投稿上传；
- **后台**：视频与专辑/分类管理、用户与站点配置、操作日志与运行环境查看。

视频文件保存在服务器本地目录（`UPLOAD_ROOT`），通过 API 流式播放，无需单独安装客户端。

### 1.2 技术环境（供了解）

- 后端：**FastAPI**；数据库常用 **SQLite**（体验/单机）或 **MySQL**（生产）。
- 前端：原生 **HTML / CSS / JavaScript**（B 站风格 UI）。
- 部署：支持 **Docker Compose**（含 **Redis**、**ffmpeg** 转码）、Nginx 反代、systemd。
- 上传 **MOV、MKV、AVI** 等格式时，服务端会自动转码为浏览器可播的 **MP4（H.264 + AAC）**（需安装 ffmpeg，Docker 镜像已内置）。

### 1.3 如何访问与登录

1. 使用 **Chrome、Edge 或 Safari** 等主流浏览器，访问站点地址（本地开发示例：`http://127.0.0.1:8808/`）。
2. 打开 **登录** 页面（`/login`），使用管理员分配的账号登录。  
   首次部署默认管理员：**admin** / **admin2026**（请尽快修改密码）。
3. 密码错误或无法登录时，请联系 **管理员** 重置；多次失败可能触发登录锁定（若已启用）。

### 1.4 登录后的界面

- **顶栏 / 侧栏导航**：视频首页、投稿、管理后台等入口（具体以站点外观配置为准）。
- **主区域**：当前功能的列表、播放器或表单。
- **个人中心**：头像、时区、修改密码等。

管理员可进入 **仪表盘、视频管理、用户管理、站点外观、系统配置、操作日志** 等页面；普通用户主要使用前台浏览与投稿，以及管理自己上传的视频。

## 三、分功能说明

### 3.1 视频首页

- 展示 **公开** 视频列表，支持关键词搜索、分类筛选。
- **隐私视频** 需登录后在对应专区或链接中查看。
- 是否允许 **游客播放** 公开视频，由站点配置 `VIDEO_ALLOW_ANONYMOUS_PLAY` 决定。

### 3.2 播放页

- 地址使用加密参数：`/video?v=<ref>`，不直接暴露数字 ID。
- 支持 HTML5 **流式播放**、断点续传；显示时长、播放量、点赞/收藏状态。
- **评论**：登录用户可发表；游客可预览前 N 条（由 `VIDEO_COMMENT_GUEST_PREVIEW` 配置）。
- **分享**：
  - 公开视频：登录用户可复制分享链接；
  - 隐私视频：可选择「需登录访问」或「限时公开访问」（上传者/管理员设置有效期）。

### 3.3 视频投稿

1. 进入 **投稿** 页，选择视频文件（支持 MP4、MOV、MKV、WebM、AVI、M4V 等）。
2. 系统 **分片上传**，支持大文件与断点续传。
3. 可选 **裁剪封面**、填写标题/简介、选择 **公开/隐私**、分类与专辑。
4. 提交后，若格式非浏览器友好（如 MOV），服务端会 **自动转码为 MP4**；转码完成前请勿关闭页面（大文件可能耗时较长）。
5. 转码成功后可立即在列表/播放页观看。

### 3.4 专辑与分类

- **分类**：管理员维护；投稿与列表可按分类筛选。
- **专辑**：将多集视频组织为合集，便于连续观看。

### 3.5 视频管理（后台）

- 列表搜索、预览、编辑标题/简介/可见性/分类。
- **替换视频文件**：重新上传后同样会走转码流程。
- **删除**：移除记录及本地文件（需相应权限）。
- 普通用户一般只能管理 **自己上传** 的视频；管理员可管理全站。

### 3.6 站点外观（管理员）

上传 Logo、Favicon，配置主题色、站点名称与页脚版权等，保存后前台即时生效。

### 3.7 系统配置与运行环境（管理员）

- 查看/导出当前 **`.env` 运行参数**（密钥等敏感项会脱敏展示）。
- 查看服务器基础指标。
- 具体业务参数仍以 `.env` 与 **[deploy.md](./deploy.md)** 为准。

### 3.8 用户管理（管理员）

新增用户、分配角色（管理员/普通用户）、重置密码等。

### 3.9 操作日志（管理员）

查看登录、视频创建/删除、配置变更等审计记录，便于排查问题。

## 四、按角色使用

### 4.1 管理员

- 维护 **用户、分类、站点外观、系统配置**。
- 管理 **全站视频、专辑、评论**（删除等）。
- 查看 **仪表盘、操作日志、运行环境**。
- 配置生产环境：**SECRET_KEY**、数据库、备份邮件、Nginx 等（见部署文档）。

### 4.2 普通用户（UP 主 / 观众）

- **观看**：公开视频；登录后可看隐私专区、点赞收藏、评论、分享。
- **投稿**：上传视频、维护自己视频的封面与信息。
- **个人中心**：修改头像、密码、显示时区等。

## 五、推荐使用顺序

```text
登录 → 浏览/播放 →（可选）投稿上传 → 等待转码完成 → 分享链接
```

| 步骤 | 操作 | 页面 |
|------|------|------|
| 1 | 登录账号 | `/login` |
| 2 | 浏览或搜索视频 | `/` |
| 3 | 播放、互动 | `/video?v=...` |
| 4 | 上传新视频 | `/video_upload` |
| 5 | 管理自己的视频 | `/video_manage` |

## 六、常见问题（FAQ）

**Q1：上传 MOV 后很久才能播放？**  
系统正在 **转码为 MP4**，文件越大耗时越长。请确认服务器已安装 **ffmpeg**（Docker 部署已包含）。

**Q2：提示视频尚未就绪？**  
多为转码未完成或文件缺失，请稍后刷新或联系管理员查看日志。

**Q3：游客无法播放？**  
站点可能关闭了游客播放（`VIDEO_ALLOW_ANONYMOUS_PLAY=false`），请先 **登录**。

**Q4：隐私视频如何分享给他人？**  
在播放页使用 **分享**，选择需登录或限时公开链接。

**Q5：上传失败或 413？**  
检查文件是否超过 `VIDEO_MAX_UPLOAD_MB`；公网还需调大 Nginx `client_max_body_size`。

**Q6：忘记密码？**  
联系管理员在 **用户管理** 中重置。

**Q7：健康检查里 redis 显示 down？**  
Docker 部署应随应用启动 Redis；检查 `docker compose ps` 与 `.env` 中 `REDIS_URL`（Compose 内默认 `redis://redis:6379/0`）。

## 七、小结

1. **前台** 负责看片、互动与投稿；**后台** 负责内容与用户运营。  
2. 播放链接使用 **加密 ref**，分享时注意隐私与有效期。  
3. **非 MP4 或编码不兼容** 的上传会自动转码，需 ffmpeg 支持。  
4. **管理员** 负责站点配置与安全；**普通用户** 聚焦观看与投稿。

若本手册与当前部署版本界面有差异，以线上系统与管理员说明为准。
