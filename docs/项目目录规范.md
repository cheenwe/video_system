# 项目目录规范

本文描述 **video_system（V视频站）** 当前仓库布局与分层约定，便于协作与 Code Review。

## 1. 根目录结构

```text
video_system/
├── src/                    # Python 后端
│   ├── main.py             # FastAPI 入口、静态页挂载、/api/health
│   ├── api/                # 路由（auth、video、dashboard、site_branding …）
│   ├── models/             # SQLAlchemy ORM
│   ├── schemas/            # Pydantic 请求/响应模型
│   ├── services/           # 业务逻辑（上传、转码、播放、评论 …）
│   ├── core/               # 配置、数据库、鉴权、Redis、安全中间件
│   ├── jobs/               # 定时任务（数据库备份邮件等）
│   └── tests/              # pytest 单元测试
├── web/                    # 静态前端（HTML / CSS / JS）
├── scripts/                # init_data、构建脚本、Docker APT 镜像脚本等
├── deploy/                 # Nginx、systemd 示例配置
├── docs/                   # 部署与用户文档
├── alembic/                # 数据库迁移（env.py、versions）
├── uploads/                # 默认上传根（视频、封面、头像、品牌资源）
├── data/                   # 默认 SQLite 数据目录（本地/Docker 卷）
├── docker-compose.yml      # app / mysql / redis / nginx 编排
├── Dockerfile              # 标准镜像（MySQL 客户端、gcc、ffmpeg）
├── Dockerfile.slim         # 轻量镜像（SQLite / 外部 MySQL）
├── requirements.txt
├── .env.example
└── README.md
```

## 2. 分层职责

| 层级 | 目录 | 职责 |
|------|------|------|
| 接口 | `src/api/` | 参数校验、依赖注入、调用 service、写操作日志 |
| 业务 | `src/services/` | 领域规则；不直接处理 HTTP |
| 模型 | `src/models/` | 表结构与关系 |
| 契约 | `src/schemas/` | API 入参/出参类型 |
| 基础设施 | `src/core/` | `config`、`database`、`deps`、`redis_client`、`exceptions` |
| 前端 | `web/` | 页面与 `web/js/*` 模块，通过 `api.js` 调后端 |

## 3. 上传与视频文件布局

所有用户文件落在 `UPLOAD_ROOT`（默认 `uploads/`）下：

| 子路径 | 内容 |
|--------|------|
| `videos/files/` | 合并/转码后的视频（一般为 `{upload_id}.mp4`） |
| `videos/chunks/` | 分片上传临时目录（合并后删除） |
| `videos/covers/` | 视频封面 JPEG |
| `user_avatars/` | 用户头像 |
| `site_branding/` | Logo、Favicon、branding.json |
| `db_backups/` | 可选本地数据库备份 ZIP |

路径解析与防穿越见 `src/core/local_storage.py`。

## 4. 关键服务模块

| 模块 | 文件 | 说明 |
|------|------|------|
| 分片上传 | `video_upload_service.py` | 初始化、续传、合并、创建视频记录 |
| 转码 | `video_transcode_service.py` | ffmpeg/ffprobe；MOV 等 → MP4 |
| 播放与权限 | `video_service.py` | 列表、详情、流式读取、可见性 |
| 加密引用 | `video_ref_service.py` | 播放链接 `v=` 参数 |
| 封面 | `video_cover_service.py` | 裁剪图保存与 URL |
| 互动 | `video_interaction_service.py` | 点赞、收藏 |
| 评论 | `video_comment_service.py` | 播放页评论 |
| 分类/专辑 | `video_taxonomy_service.py` | 分类与合集 |

## 5. 配置约定

- 运行参数统一从 **`.env`** 加载，定义在 `src/core/config.py` 的 `Settings`。
- 管理端「运行环境」页说明见 `src/core/runtime_env_docs.py`。
- Docker Compose 可覆盖 `DATABASE_URL`、`REDIS_URL` 等；构建参数 `APT_MIRROR` 控制 Debian 源（默认华为云镜像）。

## 6. 测试与质量

- 单元测试放在 `src/tests/`，命名 `test_*.py`。
- 提交前建议：`pytest -q`、`python -m compileall -q src`（与 CI 一致）。
